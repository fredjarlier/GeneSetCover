Use to prepare sending/receiving "Reads" for each process. We will know the order of the most stable iteration of send/recv between Process We will know for each process how many Reads and Clusters it will send to/rcv from others.

Parameters
[out]nbread_send_by_procArray of size_t which will contain how many Reads, process local will send to others
[out]nbread_rcv_by_procArray of size_t which will contain how many Reads, process local will receive from others
[out]nbcluster_send_by_procArray of size_t which will contain how many Clusters, process local will send to others
[out]nbcluster_rcv_by_procArray of size_t which will contain how many Clusters, process local will receive from others
[out]cs_updateLinked list of Clusterplit. Use to know where send a cluster.
[out]pClusterLinked list of Clusters local (cs_update and pCluster must have same clusters in the same order )
[out]order_iterationArray organized by most stable Iteration first
[in]rankProcess rank
[in]num_procNumber of process
Returns
